--- Docker Compose деплой ---

Создана полная Docker-инфраструктура:
  - Dockerfile (Python 3.14-slim + Gunicorn)
  - docker-compose.yml (production: Nginx + Django + PostgreSQL + Redis)
  - docker-compose.dev.yml (development: Django + SQLite)
  - nginx.conf (обратный прокси, раздача статики/медиа)
  - entrypoint.sh (миграции + collectstatic + запуск)
  - .env.example / .env.dev (переменные окружения)
  - .dockerignore / .gitignore

ТРУДНОСТИ И РЕШЕНИЯ:

1. Вложенный .git в директории ORANGE/
   Проблема: Docker игнорирует директории с .git при сборке контекста.
   Решение: Удалён вложенный .git, все файлы ORANGE/ добавлены в основной
            репозиторий как обычная директория.

2. Неправильные пути COPY в Dockerfile
   Проблема: COPY ./ORANGE/ не работал, т.к. контекст сборки — корень проекта.
   Решение: Изменено на COPY ORANGE/ (без ./).

3. .dockerignore исключал ORANGE/
   Проблема: Файлы проекта не попадали в образ.
   Решение: Убрано исключение ORANGE/ из .dockerignore.

4. Rate limit Docker Hub
   Проблема: "unauthenticated pull rate limit" при скачивании образов.
   Решение: docker login (бесплатный аккаунт, лимит 200 pull/6ч).

5. ALLOWED_HOSTS
   Проблема: DisallowedHost при доступе по IP сервера (72.56.249.17).
   Решение: Добавлен IP в DJANGO_ALLOWED_HOSTS в .env.dev.

6. source.unsplash.com закрыт
   Проблема: Все изображения через source.unsplash.com перестали работать
            (сервис закрыт в 2024 году).
   Решение: Заменены на CSS-градиентные заглушки с эмодзи.

7. 502 Bad Gateway
   Проблема: Nginx работал, но Django-контейнер отсутствовал.
   Причина: Запущен production compose, но django упал.
   Решение: docker compose down && docker compose up --build -d

8. ГЛАВНАЯ ПРОБЛЕМА: сайт отображался только текстом (без стилей)
   Проблема: CSS и JS файлы загружались (HTTP 200), но стили не применялись.
   Причина: В nginx.conf отсутствовала директива include mime.types.
            Nginx отдавал CSS как application/octet-stream вместо text/css.
            Браузер игнорировал файл, т.к. не распознал его как таблицу стилей.
   Решение: Добавлено в nginx.conf:
              http {
                  include /etc/nginx/mime.types;
                  default_type application/octet-stream;
                  ...
              }
   УРОК НА БУДУЩЕЕ: При создании любого nginx.conf ВСЕГДА включать mime.types.

9. Дубликаты статики при collectstatic
   Предупреждение: "Found another file with the destination path..."
   Причина: Volume staticfiles содержал старые файлы, которые конфликтовали
            с новыми.
   Решение: docker compose down -v (удаление volume) + пересборка.

10. Коммит не попал на GitHub
    Проблема: Локальный коммит f931abe не был отправлен на origin.
    Причина: Предыдущий git push не выполнился (нужна авторизация).
    Решение: Повторный git push origin master.
